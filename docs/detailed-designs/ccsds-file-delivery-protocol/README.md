# CCSDS File Delivery Protocol — detailed design

The FDP subsystem exposes CFDP-compatible file transfer operations (CCSDS 727.0-B, single declared baseline profile per TBD-008) through Tranquility's public API. HTTP requests at `POST /api/filetransfer/{instance}/{serviceName}/transfers` (and the `:pause` / `:resume` / `:cancel` action methods) are routed through the ASP.NET Core HTTP API into application-layer CQRS handlers, which resolve the target instance's file transfer service via the Instance Registry and drive the core-domain CFDP Engine. The engine owns per-transfer CFDP Transaction state machines that emit and consume Metadata, File Data, EOF, ACK, and Finished PDUs over the PDU Router and UDP Data Link toward the ground station and spacecraft, reading and writing file content in the Object Store and persisting transfer records in the Transfer Archive. Transfer state changes — progress, completion, and failure — are fanned out by the Subscription Manager to WebSocket `file-transfers` subscribers, and a CI-driven conformance flow inspects that every baseline conformance test references the one declared CFDP profile.
