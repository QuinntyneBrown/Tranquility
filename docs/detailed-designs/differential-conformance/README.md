# Differential Conformance — detailed design

The Differential Conformance (DIF) subsystem is a test harness that proves Tranquility's interoperability and clean-room fidelity by replaying a recorded telemetry/command corpus, byte-identically and in order, into both a Tranquility instance and a stock reference mission-control deployment, then comparing externally observable outputs across six required surfaces (parameter values, engineering conversions, timestamps, alarm states, command history, API responses). A Harness Orchestrator in the application layer drives three stages — parallel corpus ingest via link adapters, external output capture via the published HTTP/WebSocket APIs of both systems, and per-surface comparison by a Comparison Engine — persisting equivalence and divergence records to report and divergence stores. Detected divergences are triaged through the HTTP API into one of three defined classes (Tranquility defect, reference behavior outside standard, standard ambiguity), and clean-room integrity is enforced by a CI-driven manifest scan plus design-review inspection confirming the harness observes external behavior only and requires no reference source-code artifacts.
