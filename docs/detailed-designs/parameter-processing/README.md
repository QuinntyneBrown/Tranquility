# Parameter Processing — detailed design

The Parameter Processing (PAR) subsystem turns decommutated raw telemetry samples into monitored, publishable parameter values. A Telemetry Processor in the application layer orchestrates a core-domain pipeline driven entirely by mission database (MDB/XTCE) definitions: the Calibration Engine converts raw values to engineering values using the active calibrator for each parameter, the Derived Parameter Evaluator executes MDB algorithms whose inputs changed, the Limit Monitor evaluates each engineering value against configured alarm ranges and yields a monitoring result, and the Alarm Manager detects monitoring-level transitions and raises alarm notifications. Processed samples — carrying both `acquisitionTime` and `generationTime` — are stored in the Parameter Cache and fanned out by the Subscription Manager to operator clients over the WebSocket API.
