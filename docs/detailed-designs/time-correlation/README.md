# Time Correlation — detailed design

The Time Correlation (TIM) subsystem maps onboard spacecraft time to UTC and enforces a single declared CCSDS 301.0-B time-code profile for all mission time decode/encode behavior. A per-instance Time Correlation Service (TCO) accumulates (onboard-time, ground-time) sample pairs from downlinked TM frames — correcting earth reception time by a time-of-flight estimate — fits linear coefficients (gradient, offset) over the sample buffer, and monitors deviation against configured accuracy/validity thresholds. Operators observe and control the service through the documented HTTP API: a status endpoint exposes coefficients, sample state, and deviation; set-config and set-coefficients operations perform authorized, audited updates to the correlation model; and time-of-flight interval add/delete operations manage the one-way-delay interval set used for ground-time correction. All mutating operations pass through an authorization filter (ControlTimeCorrelation privilege), CQRS command handlers in the application layer, and the instance/processor registry to resolve the target service; a conformance harness verifies decode/encode round-trip fidelity of the declared time-code profile in CI.
