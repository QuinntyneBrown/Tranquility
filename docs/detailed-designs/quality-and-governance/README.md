# Quality and Governance — detailed design

The Quality and Governance (QLT) subsystem is not a runtime feature but a set of enforced gates and reviews that keep the Tranquility baseline trustworthy: CI pipeline stages verify platform metadata (.NET LTS, Linux-first), dependency license policy (Apache-2.0 compatibility), deterministic behaviour of the decommutation core inside an I/O-free isolation boundary, and CQRS command/query interface separation in the application layer; complementary human-in-the-loop inspections confirm that the active baseline declares explicit numeric performance targets, that measured benchmark evidence satisfies those targets, and that deferred capability areas (SDLS, expanded PUS coverage, HA replication) remain outside the baseline unless formally rebaselined. Each flow below produces persisted, revision-tagged evidence (audit reports, verification records, inspection sign-offs) so that every governance decision is traceable, and each gate has an explicit failure path that blocks merge or release promotion.

| Diagram | Behaviour | Requirements |
|---|---|---|
| [release-platform-metadata-inspection.puml](release-platform-metadata-inspection.puml) | Release metadata inspected for current .NET LTS target and Linux-first OS support; non-compliant metadata fails the platform gate | L1-QLT-001, L2-QLT-001 |
| [decomm-core-determinism-verification.puml](decomm-core-determinism-verification.puml) | Static boundary analysis plus repeated isolated execution proves the decommutation core is deterministic and free of network/filesystem dependencies; divergence or boundary breach fails the stage | L1-QLT-001, L2-QLT-002 |
| [license-audit-nominal.puml](license-audit-nominal.puml) | CI license audit resolves the full dependency graph, classifies every license against the Apache-2.0 policy, archives the report, and passes the build | L1-QLT-001, L2-QLT-003 |
| [license-audit-violation.puml](license-audit-violation.puml) | A disallowed license in the dependency graph fails the pipeline, blocks merge/release promotion, and records the offending package | L1-QLT-001, L2-QLT-003 |
| [performance-baseline-declaration-review.puml](performance-baseline-declaration-review.puml) | Inspection of the active baseline revision verifies each required performance metric has an explicit numeric target; missing or non-numeric targets reject the revision | L1-QLT-002, L2-QLT-004 |
| [performance-target-verification.puml](performance-target-verification.puml) | Benchmark harness drives ingest, processing, streaming, and archive under load, compares measured values against declared numeric targets, and archives signed verification evidence; a missed target fails the perf stage | L1-QLT-002, L2-QLT-004 |
| [scope-control-trace-review.puml](scope-control-trace-review.puml) | Baseline requirement trace review confirms deferred capability areas (SDLS, expanded PUS, HA replication) have no implementation requirements; violations force formal scope revision or baseline correction | L1-QLT-002, L2-QLT-005 |
| [cqrs-conformance-analysis.puml](cqrs-conformance-analysis.puml) | Architecture conformance analysis inspects application service interfaces to verify mutation and retrieval paths are separated by contract; mixed handlers fail the stage | L1-QLT-001, L2-QLT-006 |
