# Lifecycle subsystem — L2 requirements

## L2-LIF-001

- **Requirement:** Tranquility shall provide instance discovery and detail retrieval through documented Instances methods.
- **Rationale:** Client startup and operator workflows depend on instance introspection.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-LIF-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN at least one configured instance WHEN instances methods are called THEN state and metadata are returned in documented structure.

## L2-LIF-002

- **Requirement:** Tranquility shall provide processor lifecycle operations compatible with documented processing methods.
- **Rationale:** Replay and real-time processing contexts must be administratively controllable.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Demonstration
- **Parent:** L1-LIF-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN processor lifecycle requests WHEN create/edit/delete/list methods are executed THEN processor state reflects requested operation.

## L2-LIF-003

- **Requirement:** Tranquility shall support the documented persistent processor option semantics.
- **Rationale:** Mission operations may require processors that survive session-level lifecycle events.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-LIF-001
- **Priority:** Should
- **Acceptance Criteria:**
  - GIVEN a processor created with `persistent=true` WHEN processor lifecycle transitions occur THEN persistence behavior matches documented semantics.

## L2-LIF-004

- **Requirement:** Tranquility shall expose replay state transitions through documented processor state fields.
- **Rationale:** Replay observability is required for archive analysis workflows.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Inspection
- **Parent:** L1-LIF-001
- **Priority:** Should
- **Acceptance Criteria:**
  - GIVEN a replay processor WHEN replay starts, pauses, and stops THEN exposed replay-state fields reflect those transitions.
