# Security subsystem — L2 requirements

## L2-SEC-001

- **Requirement:** Tranquility shall implement documented IAM resource operations for users, groups, roles, and service accounts.
- **Rationale:** Administrative interoperability depends on IAM resource compatibility.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-SEC-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN an authenticated administrator WHEN IAM list/create/update methods are called THEN responses follow documented IAM resource structures.

## L2-SEC-002

- **Requirement:** Tranquility shall require authenticated identity for API operations that mutate server state.
- **Rationale:** Unauthenticated mutation is incompatible with mission security posture.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-SEC-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN an unauthenticated caller WHEN a state-changing API operation is requested THEN the operation is rejected.

## L2-SEC-003

- **Requirement:** Tranquility shall enforce authorization checks for command issue, queue action, and link control methods.
- **Rationale:** Uplink and link controls are privileged operations.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-SEC-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN an authenticated user lacking required privilege WHEN privileged control methods are called THEN requests are denied with authorization error.

## L2-SEC-004

- **Requirement:** Tranquility shall record immutable audit entries for authentication events, authorization denials, and command uplink actions.
- **Rationale:** Security incident response and compliance evidence require complete audit trails.
- **Source:** External API documentation package (URL redacted), User brief §9
- **Verification:** Inspection
- **Parent:** L1-SEC-002
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN successful or denied privileged operations WHEN audit is queried THEN corresponding records include actor, action, timestamp, and outcome.

## L2-SEC-005

- **Requirement:** Tranquility shall provide TLS protection for HTTP and WebSocket API transport.
- **Rationale:** Command and telemetry interfaces require encrypted transport in production.
- **Source:** User brief §9, External API documentation package (URL redacted)
- **Verification:** Demonstration
- **Parent:** L1-SEC-002
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a production deployment WHEN clients connect to API endpoints THEN transport is established over TLS.
