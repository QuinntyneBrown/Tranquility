# Space Packet Processing and Decommutation subsystem — L2 requirements

## L2-SPP-001

- **Requirement:** Tranquility shall decode the CCSDS space packet primary header fields required for packet routing and validity checks.
- **Rationale:** Header decoding is required to identify packet type and route processing.
- **Source:** CCSDS 133.0-B (clause mapping pending: TBD-001)
- **Verification:** Test
- **Parent:** L1-SPP-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a valid space packet WHEN parsed THEN header fields used by routing are extracted with expected values.

## L2-SPP-002

- **Requirement:** Tranquility shall route parsed packets into XTCE container resolution using active mission database definitions.
- **Rationale:** Decommutation must be model-driven to remain mission-configurable.
- **Source:** OMG XTCE 1.3 (clause mapping pending: TBD-010)
- **Verification:** Test
- **Parent:** L1-SPP-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a packet mapped in MDB WHEN decommutation executes THEN the expected container and parameter set is produced.

## L2-SPP-003

- **Requirement:** Tranquility shall produce deterministic decommutation output for identical input packet bytes and identical MDB version.
- **Rationale:** Determinism is required for differential conformance and reproducible verification.
- **Source:** User brief §9 (deterministic, testable decommutation core)
- **Verification:** Analysis
- **Parent:** L1-SPP-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN repeated runs of the same packet corpus with the same MDB WHEN outputs are compared THEN extracted values and timestamps are identical.

## L2-SPP-004

- **Requirement:** Tranquility shall report packet parse failures as explicit processing errors.
- **Rationale:** Failure transparency is required for operations and safety-case evidence.
- **Source:** User brief §9 (tight error handling)
- **Verification:** Demonstration
- **Parent:** L1-SPP-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a packet with invalid structural fields WHEN parse is attempted THEN a processing error is emitted and contains a machine-readable reason.
