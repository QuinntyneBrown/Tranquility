# Real-time streaming subsystem — L2 requirements

## L2-RTS-001

- **Requirement:** Tranquility shall implement WebSocket call correlation fields compatible with documented `reply`, `call`, and `seq` semantics.
- **Rationale:** Subscription clients depend on call/sequence semantics for multiplexed subscriptions.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-RTS-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN concurrent subscriptions WHEN server messages are received THEN each message carries call/seq fields enabling deterministic correlation.

## L2-RTS-002

- **Requirement:** Tranquility shall support the documented `parameters` subscription topic behavior for parameter updates.
- **Rationale:** Parameter subscriptions are a primary real-time client workload.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-RTS-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a valid parameters subscription request WHEN telemetry arrives THEN parameter updates are emitted on the subscription stream.

## L2-RTS-003

- **Requirement:** Tranquility shall support documented subscription topics for processor and link state updates.
- **Rationale:** Mission clients need push updates for processor/link operational state.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Demonstration
- **Parent:** L1-RTS-001
- **Priority:** Should
- **Acceptance Criteria:**
  - GIVEN active processor/link subscriptions WHEN runtime state changes THEN state updates are emitted without polling.

## L2-RTS-004

- **Requirement:** Tranquility shall expose frame-drop detectability through documented sequence progression behavior.
- **Rationale:** Clients need to detect missed updates under load.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Analysis
- **Parent:** L1-RTS-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN an induced drop condition WHEN subscription traffic is analyzed THEN sequence discontinuity is observable by the client.
