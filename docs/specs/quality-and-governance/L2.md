# Quality and governance subsystem — L2 requirements

## L2-QLT-001

- **Requirement:** Tranquility shall target the current .NET LTS runtime with Linux-first deployment support.
- **Rationale:** Platform consistency is required for mission deployment and maintenance planning.
- **Source:** User brief §9 (2026-07-15)
- **Verification:** Inspection
- **Parent:** L1-QLT-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN release metadata WHEN reviewed THEN target runtime and OS support include current .NET LTS and Linux.

## L2-QLT-002

- **Requirement:** Tranquility shall maintain a deterministic decommutation core boundary that is independent from runtime I/O concerns.
- **Rationale:** Deterministic, side-effect-free core behavior is required for provable verification.
- **Source:** User brief §9 (2026-07-15)
- **Verification:** Analysis
- **Parent:** L1-QLT-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN decommutation test inputs WHEN executed in isolation THEN outputs are reproducible without network or filesystem dependencies.

## L2-QLT-003

- **Requirement:** Tranquility shall execute dependency license audits in CI and fail builds on policy violations.
- **Rationale:** Apache-2.0 distribution goals require enforceable dependency-license control.
- **Source:** User brief §9 (2026-07-15), TBD-020
- **Verification:** Test
- **Parent:** L1-QLT-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a dependency graph containing a disallowed license WHEN CI license audit runs THEN pipeline status is failed.

## L2-QLT-004

- **Requirement:** Tranquility shall declare numeric baseline targets for ingest rate, parameter update rate, end-to-end latency percentiles, and archive write throughput.
- **Rationale:** Quantified targets are required to define acceptance for performance work.
- **Source:** User brief §9 (2026-07-15), TBD-013
- **Verification:** Inspection
- **Parent:** L1-QLT-002
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN the active baseline revision WHEN performance section is reviewed THEN each required metric has an explicit numeric target.

## L2-QLT-005

- **Requirement:** Tranquility shall keep SDLS, expanded PUS coverage, and HA replication outside this baseline unless scope is formally revised.
- **Rationale:** Scope control prevents accidental expansion beyond approved baseline.
- **Source:** User brief §5 (deferred items)
- **Verification:** Inspection
- **Parent:** L1-QLT-002
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN baseline requirement review WHEN features are traced THEN deferred capability areas have no implementation requirements in this baseline.

## L2-QLT-006

- **Requirement:** Tranquility shall maintain a distinct command-side mutation interface and query-side retrieval interface in its application architecture.
- **Rationale:** Required architecture constraints include CQRS-oriented separation for clarity and testability.
- **Source:** User brief §9 (Clean Architecture and CQRS)
- **Verification:** Analysis
- **Parent:** L1-QLT-001
- **Priority:** Should
- **Acceptance Criteria:**
  - GIVEN architecture conformance review WHEN service interfaces are inspected THEN mutation paths and retrieval paths are separated by interface contract.
