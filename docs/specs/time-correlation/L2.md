# Time correlation subsystem — L2 requirements

## L2-TIM-001

- **Requirement:** Tranquility shall implement `GET /api/tco/{instance}/{serviceName}/status` with documented status fields.
- **Rationale:** Operators need visibility into current correlation coefficients and deviation state.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-TIM-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a configured TCO service WHEN status endpoint is called THEN returned payload includes coefficient and sample status fields.

## L2-TIM-002

- **Requirement:** Tranquility shall provide documented TCO configuration and coefficient update operations.
- **Rationale:** Missions require controlled updates to time-correlation model parameters.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Demonstration
- **Parent:** L1-TIM-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a valid configuration change WHEN set-config or set-coefficients methods are invoked THEN subsequent status reflects updated values.

## L2-TIM-003

- **Requirement:** Tranquility shall provide documented management of time-of-flight interval records.
- **Rationale:** End-to-end timing correction requires explicit one-way delay interval control.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-TIM-001
- **Priority:** Should
- **Acceptance Criteria:**
  - GIVEN an interval add or delete request WHEN method executes THEN interval set changes are visible in time-correlation service state.

## L2-TIM-004

- **Requirement:** Tranquility shall apply one declared CCSDS time-code profile for mission time decode/encode behavior.
- **Rationale:** A fixed profile is needed for deterministic time handling and conformance tests.
- **Source:** CCSDS 301.0-B (profile/clause mapping pending: TBD-009)
- **Verification:** Analysis
- **Parent:** L1-TIM-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN mission time values encoded per declared profile WHEN decoded and re-encoded THEN round-trip behavior matches profile rules.
