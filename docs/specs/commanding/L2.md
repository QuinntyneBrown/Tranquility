# Commanding subsystem — L2 requirements

## L2-CMD-001

- **Requirement:** Tranquility shall support command issue through `POST /api/processors/{instance}/{processor}/commands/{name}` with documented request/response fields.
- **Rationale:** Command issue compatibility is required for client portability.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-CMD-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a valid command payload WHEN posted to the command issue URI THEN response includes command identifier and generated binary fields.

## L2-CMD-002

- **Requirement:** Tranquility shall expose command queue inspection and command accept/reject operations through documented queue methods.
- **Rationale:** Operational command release control depends on queue workflows.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Demonstration
- **Parent:** L1-CMD-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN queued commands WHEN queue API methods are invoked THEN queue state and entries are returned and accept/reject actions update queue contents.

## L2-CMD-003

- **Requirement:** Tranquility shall implement COP-1 behavior according to the selected mission profile.
- **Rationale:** Reliable telecommand transfer requires profile-conformant COP-1 handling.
- **Source:** CCSDS 232.1-B (profile/clause mapping pending: TBD-006), External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-CMD-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a COP-1 session configured to mission defaults WHEN command transfer executes THEN observed protocol state transitions follow the selected profile.

## L2-CMD-004

- **Requirement:** Tranquility shall produce CLTU output according to the selected mission coding/synchronization profile.
- **Rationale:** Link-layer command transmission requires CLTU compatibility.
- **Source:** CCSDS 231.0-B (profile/clause mapping pending: TBD-007)
- **Verification:** Test
- **Parent:** L1-CMD-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a validated TC frame WHEN CLTU generation executes THEN output conforms to the selected CLTU profile.

## L2-CMD-005

- **Requirement:** Tranquility shall persist command history entries for each command lifecycle stage exposed by the API.
- **Rationale:** Post-pass analysis and anomaly triage require lifecycle traceability.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Inspection
- **Parent:** L1-CMD-002
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a command that is queued and released WHEN history is queried THEN lifecycle records show ordered stage progression.

## L2-CMD-006

- **Requirement:** Tranquility shall require elevated privilege for command issue options marked as privileged in the documented API.
- **Rationale:** Override capabilities must be access-controlled.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-CMD-002
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a caller without elevated privilege WHEN privileged command options are submitted THEN the request is denied with authorization error.
