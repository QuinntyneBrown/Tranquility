# Archive subsystem — L2 requirements

## L2-ARC-001

- **Requirement:** Tranquility shall provide parameter history retrieval at `GET /api/archive/{instance}/parameters/{name}` with documented query controls.
- **Rationale:** Historical value retrieval is required for analysis and incident timelines.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-ARC-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN archived parameter data WHEN history endpoint is queried with time bounds THEN returned values are bounded by requested interval.

## L2-ARC-002

- **Requirement:** Tranquility shall provide server-streaming parameter replay at `POST /api/archive/{instance}:streamParameterValues`.
- **Rationale:** Replay streams are required for analysis workflows that process long intervals.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Demonstration
- **Parent:** L1-ARC-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a replay request with start/stop times WHEN stream endpoint is invoked THEN parameter data is emitted as a chunked server stream.

## L2-ARC-003

- **Requirement:** Tranquility shall provide archived segment listing at `GET /api/parameter-archive/{instance}/pids/{pid}/segments`.
- **Rationale:** Segment-level introspection supports archive coverage diagnostics and tooling.
- **Source:** External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-ARC-001
- **Priority:** Should
- **Acceptance Criteria:**
  - GIVEN a valid parameter ID WHEN segment endpoint is queried THEN segment metadata is returned with start, end, and count fields.

## L2-ARC-004

- **Requirement:** Tranquility shall support replay-based retrieval mode where documented API behavior specifies replay as a source option.
- **Rationale:** Replay source mode is needed when archive coverage is incomplete or reprocessing is required.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** L1-ARC-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a history request with replay source option WHEN processed THEN data is returned through replay-mode processing with active MDB semantics.
