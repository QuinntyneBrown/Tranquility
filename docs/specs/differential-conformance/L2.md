# Differential conformance subsystem — L2 requirements

## L2-DIF-001

- **Requirement:** Tranquility shall ingest an identical recorded input corpus in parallel with the reference system during differential test execution.
- **Rationale:** Input equivalence is required before any output equivalence claim is meaningful.
- **Source:** User brief §8 (2026-07-15)
- **Verification:** Test
- **Parent:** L1-DIF-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a recorded telemetry/command corpus WHEN differential harness runs THEN both systems consume the same corpus bytes in the same order.

## L2-DIF-002

- **Requirement:** Tranquility shall compare parameter values, engineering conversions, timestamps, alarm states, command history, and API responses against reference outputs.
- **Rationale:** Conformance confidence requires multi-surface comparison, not single-metric checks.
- **Source:** User brief §8 (2026-07-15)
- **Verification:** Analysis
- **Parent:** L1-DIF-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN completed differential execution WHEN reports are generated THEN each required output surface has an equivalence or divergence record.

## L2-DIF-003

- **Requirement:** Tranquility shall classify each detected divergence into one of the defined triage classes.
- **Rationale:** Triage classes drive corrective action and standards clarification.
- **Source:** User brief §8 (2026-07-15)
- **Verification:** Inspection
- **Parent:** L1-DIF-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN a divergence entry WHEN triage is completed THEN classification is one of: Tranquility defect, reference behavior outside standard, or standard ambiguity.

## L2-DIF-004

- **Requirement:** Tranquility shall ensure the differential harness uses external behavior observation only.
- **Rationale:** Clean-room integrity requires avoiding reference source-code dependence.
- **Source:** User brief §2.1 and §8 (2026-07-15)
- **Verification:** Inspection
- **Parent:** L1-DIF-001
- **Priority:** Must
- **Acceptance Criteria:**
  - GIVEN harness design review WHEN evidence is inspected THEN no reference source-code artifacts are required by the harness.
