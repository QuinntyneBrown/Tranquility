# API subsystem — L1 requirements

## L1-API-001

- **Requirement:** Tranquility shall expose HTTP and WebSocket interfaces that are wire-compatible with the documented reference mission-control system API.
- **Rationale:** Compatibility enables existing mission clients to switch server back ends without client rewrites.
- **Source:** External API documentation package (URL redacted), User brief §7.1 (2026-07-15)
- **Verification:** Demonstration
- **Parent:** TRQ-HLR
- **Priority:** Must

## L1-API-002

- **Requirement:** Tranquility shall preserve documented API wire semantics for error payloads, timestamp encoding, and message framing.
- **Rationale:** Client interoperability depends on exact wire-level conventions, not only endpoint presence.
- **Source:** External API documentation package (URL redacted), External API documentation package (URL redacted)
- **Verification:** Test
- **Parent:** TRQ-HLR
- **Priority:** Must
