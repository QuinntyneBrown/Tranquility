# Support

## Getting help

Before asking for help:

1. Review the [README](README.md), [development conventions](DEVELOPMENT.md),
   [specification index](docs/specs/README.md), and
   [detailed designs](docs/detailed-designs/README.md).
2. Search [existing issues](https://github.com/QuinntyneBrown/Tranquility/issues)
   for the same problem.
3. Confirm the issue still occurs on the latest `main` branch with the SDK
   pinned in [global.json](global.json).
4. Run the relevant build or test command and retain sanitized output.

For reproducible defects, open a
[bug report](https://github.com/QuinntyneBrown/Tranquility/issues/new?template=bug_report.yml).
For scoped improvements, open a
[feature request](https://github.com/QuinntyneBrown/Tranquility/issues/new?template=feature_request.yml).

Include expected and actual behavior, reproduction steps, relevant operating
system and .NET SDK details, affected requirement IDs, and sanitized logs.

Never post credentials, signing keys, connection strings, mission data, private
interface documents, or security vulnerabilities.

## Security and conduct

- Report vulnerabilities privately according to [SECURITY.md](SECURITY.md).
- Report conduct concerns through the private channel in
  [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

Support is provided by maintainers and contributors on a best-effort basis.
This repository does not provide a service-level agreement, mission operations
support, or emergency response service.
